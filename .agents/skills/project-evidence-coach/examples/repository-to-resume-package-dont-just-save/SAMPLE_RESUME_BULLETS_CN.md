# 06_RESUME_BULLETS_CN.md 示例

## A. 保守版：确认 ownership 后可直接放简历

- 设计并制作「别只收藏」产品经理作品集 Demo，围绕内容创作者“收藏后难以转化为视频创作任务”的痛点，梳理从保存内容、补充创作启发、生成任务卡到复盘筛选的核心流程。
- 基于小样本探索调研完成早期问题判断，提炼“收藏不是问题，转化才是问题”的产品定位，并将需求收敛为保存、分类、任务卡和复盘闭环。
- 搭建可交互静态原型，覆盖灵感记录、收集箱、任务池和复盘流程，并使用 `localStorage`（浏览器本地存储）模拟数据保存。

## B. AI 产品经理版：确认 ownership 后可使用

- 设计“模拟 AI 能力”验证流程，让用户补充一句创作启发后，由系统模拟识别创作用途、标签、参考价值和下一步行动，用于验证 AI 在创作前链路中的辅助价值。
- 明确 AI 能力边界：当前阶段不追求自动生成完整视频，而是优先验证“识别用途、降低启动成本、辅助转任务”的轻量场景。

## C. To C 产品经理版：确认 ownership 后可使用

- 面向新手内容创作者设计低摩擦素材收集与任务转化流程，通过“保存灵感 -> 收集箱筛选 -> 生成创作任务 -> 复盘处理”降低从收藏到行动的断点。
- 将用户收藏内容拆解为选题、标题、封面、脚本、素材和表达方式等创作用途，提升 To C 创作者工具的信息组织效率。

## D. 禁用表达：补证据前不能写

- 补证据前禁用：声称已有线上留存、转化或使用效果改善。
- 补证据前禁用：声称已接入生产级 AI 能力并取得准确率结果。
- 补证据前禁用：把小样本探索调研包装成广泛市场结论。
- 补证据前禁用：实现收入增长、付费转化或商业化闭环。
- 补证据前禁用：完成 Android/Kotlin/Room 移动端实现，除非补充对应源码或构建证据。

## Export wording

- `确认个人贡献后可使用`：证据较强，但需要确认候选人实际负责范围。
- `补证据前禁用`：缺少原始证据、真实指标、真实 AI 接入或 ownership 证明。
