# Growth File Schema

Keep the editable record in `project-evidence/PROJECT_GROWTH.md`.

## Project and target

- project name and current stage
- target role and job-description source
- deadline, available time, and constraints

## Requirement map

- normalized requirement
- source wording or location
- importance
- current support status
- relevant evidence identifiers

## Evidence ledger

- stable identifier
- supported claim or capability
- source file, link, result, or user confirmation
- temporal status
- confidence and unresolved questions
- supported outputs
- linked job requirements

## Current diagnosis

Use these maturity states:

- missing
- initial
- presentable
- verifiable
- application-ready

## Priority queue

Keep the top three gaps with short reasons and dependencies.

## Active action card

Store exactly one expanded task with acceptance criteria, expected artifact, and time budget.

## Completed actions and output readiness

Record concise changes, evidence added, diagnosis change, and outputs now supported.
