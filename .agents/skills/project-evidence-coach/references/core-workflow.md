# Core Workflow

Follow the same loop every round:

1. Parse the target role and project artifacts.
2. Inspect what exists before asking for missing material.
3. Classify evidence as existing verified evidence, partial or weak evidence, retrospective validation, or proposed future work.
4. Rank the three highest-value gaps.
5. Expand exactly one active action card.
6. On the next round, verify the returned artifact and update the growth record.

Rules:

- Keep the project to one active action.
- Prefer artifacts, links, files, and user-confirmed facts over summaries.
- Distinguish original work from later validation.
- Disclose inspection scope when the repository is large.
